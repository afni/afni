SUMA Behavior

For the most part, the behavior applies to both hemispheres when
control panels are joined (when SUMA_SameSurfCont = YES in .sumarc).
To separate the behavior to separate hemispheres, set
SUMA_SameSurfCont = NO in .sumarc.  Some behavior only applies to one
dataset.

A **surface object** _does_ have topological (=mesh) information.

A **surface dataset** does _not_ have topological information, and is
only nodewise information.  It is often represented by an overlay
(like a *.niml.dset file, or within the code SO->Overlays), to be
displayed _on_ a surface dataset.

GII datasets can be either surface object or surface dataset (or
other).  TBD about *.niml.dset, but they can certainly be surface
datasets.

The SC_Notebook variable is a widget for managing all surface objects
currently loaded into SUMA. 
* This notebook widget can encompass many different kinds of objects,
  even beyond surfaces (graphs, tracts, etc.).
* The children of the notebook widget at called pages.


A table of object configurations. 
  - : that this attribute/feature  does _not_ apply to multiple
      things, and that it explicitly needs to be turned off for other 
      objects when it is turned on for one
  / : that this attribute/feature  does _not_ apply to multiple
      things, and they are independent
  + : this attribute/feature _does_ apply to multiple things
  S : surface object
  D : surface dataset (like overlay)

A (Alpha) check box: When this is unchecked, subthreshold regions of the overlay
are purely transparent.  When it is checked, the opacity falls off as the point 
value falls below threshold.  This falloff may be quadratic (default) or linear.  
The quadratic/linear model may not be chosen on the surface control panel.  It 
must be selected with DriveSuma.  The function, SUMA_cb_AlphaOpacityFalloff_tb_toggled, 
handles the clicking of the A checkbox.  The AlphaOpacityFalloff_tb field of the 
surface controller (SUMA_X_SurfCont) structure is the widget associated with the A checkbox.

B (Box) check box.  When this is checked, an outline separates regions that are 
above, from those that are below, the threshold. The function, SUMA_cb_BoxOutlineThresh_tb_toggled, 
handles the clicking of the B checkbox.  The BoxOutlineThresh_tb field of the surface controller 
(SUMA_X_SurfCont) structure is the widget associated with the B checkbox.  When the B box is checked on, the Dsp setting (in the ShowMode field of the overlay) is changed either from Col to C&C or from XXX to Con.  These settings are represented by the SW_SurfCont_DsetViewCol, SW_SurfCont_DsetViewCaC, SW_SurfCont_DsetViewXXX and SW_SurfCont_DsetViewCon tokens respectively. When the B box is checked off, the Dsp setting is changed either from C&C to Col or from Con to XXX. 

A and B check boxes may each be positive for no more than one dataset.
Each dataset has an associated overlay.  The number of overlays is
given by SO->N_Overlays and the overlays are in SO->Overlays.

The function, SUMA_LoadDsetOntoSO, is called when a dataset is
loaded. When a new dataset is loaded, the A and B boxes are
initialized to off, as are variable opacity and threshold outlining
for the overlay associated with the dataset.

The function, SUMA_cb_SelectSwitchColPlane, switches between two datasets that have already been loaded.  The A and B check boxes are set from the values associated with the overlay (dataset) that has just been selected.
